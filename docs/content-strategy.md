## 3. 콘텐츠 생성 방법론

### 3.1 전체 워크플로우

```
[소스 수집] → [필터링/주제 선정] → [리서치 보강] → [AI 초안 생성] → [편집/퀄리티 체크] → [게시]
```

### 3.2 단계별 상세

#### Step 1: 소스 수집 (자동)

- `collect.ts`가 RSS 피드에서 최신 뉴스 자동 수집
- 수집 주기: 1일 1~2회
- 중복 제거: URL 정규화 + 제목 유사도 체크 (기 구현)
- 수집된 뉴스는 `collected_news` 테이블에 저장

#### Step 2: 필터링/주제 선정

**주제 선정 기준** (우선순위 순):

1. **실용성**: 한국 소상공인이 바로 적용할 수 있는가?
2. **시의성**: 최근 1주일 이내의 트렌드인가?
3. **검색 수요**: 한국어 검색 볼륨이 있는가?
4. **차별성**: 한국어 경쟁 콘텐츠가 부족한 영역인가?
5. **연결성**: AI 디렉토리 도구와 자연스럽게 연결되는가?

**주제 선정 제외 기준**:

- 순수 학술/연구 내용 (소상공인 실용성 없음)
- 영어권 전용 서비스 (한국에서 사용 불가)
- 과도한 기술 용어가 필요한 주제
- 이미 게시된 유사 주제 (DB 중복 체크)

#### Step 3: 리서치 보강

주제 선정 후, AI 초안 생성 전에 소스 데이터를 보강한다:

- 관련 뉴스 최소 3건 이상 수집
- 도구 리뷰인 경우: 공식 사이트, 가격표, 주요 기능 확인
- 업종별 가이드인 경우: 해당 업종의 AI 도입 사례 수집
- 한국 시장 데이터: 통계청, 중소기업중앙회 자료 참조

#### Step 4: AI 초안 생성

**프롬프트 설계 원칙**:

1. **한국어 자연스러움**: 번역체 금지, 구어체+전문성 균형
2. **SEO 최적화**:
   - 타겟 키워드를 제목, H2, 본문 초반에 자연 배치
   - H2/H3 헤딩 구조로 스캔 가능성 확보
   - 메타 디스크립션 150자 이내, 키워드 포함
3. **E-E-A-T 준수** (Google 품질 기준):
   - **Experience (경험)**: 실전 활용 시나리오, 구체적 예시 포함
   - **Expertise (전문성)**: 정확한 기술 정보, 근거 있는 주장
   - **Authoritativeness (권위)**: 공신력 있는 소스 인용
   - **Trustworthiness (신뢰)**: 가격 정보 정확, 장단점 균형 제시
4. **AI 디렉토리 연동**: 관련 도구 언급 시 디렉토리 링크 삽입
5. **CTA (Call-to-Action)**: 뉴스레터 구독, 관련 글 링크 유도

**시스템 프롬프트 구조**:

```
[역할 정의] → [작성 규칙] → [SEO 규칙] → [출력 형식]
```

- 역할: 한국 소상공인/중소기업 AI 활용 블로그 전문 에디터
- 톤: 전문적이지만 쉽게 읽히는 한국어
- 금지: 과장 표현, 근거 없는 주장, 번역체

#### Step 5: 편집/퀄리티 체크

**자동 퀄리티 체크 기준**:

| 항목 | 기준 | 통과 조건 |
|------|------|----------|
| **본문 길이** | 2,000~3,000자 (한국어) | 범위 내 |
| **헤딩 구조** | H2 최소 3개 | 있음 |
| **키워드 밀도** | 타겟 키워드 3~5회 | 범위 내 |
| **링크** | AI 디렉토리 링크 최소 2개 | 있음 |
| **메타 설명** | 150자 이내 | 범위 내 |
| **제목 길이** | 40자 이내 (한국어) | 범위 내 |
| **마크다운 유효성** | 깨진 마크다운 없음 | 유효 |
| **원본성** | 기존 포스트와 제목/내용 중복 없음 | 중복 없음 |

**수동 편집 체크리스트** (CEO 검수 시):

- [ ] 정보가 정확한가? (가격, 기능, URL)
- [ ] 한국어가 자연스러운가? (번역체 없는가?)
- [ ] 소상공인 관점에서 실용적인가?
- [ ] 과장이나 허위 주장이 없는가?
- [ ] CTA가 적절한가?

#### Step 6: 게시

- `publish-blog.ts`로 DB INSERT (published=true)
- ISR(1분)이 자동으로 블로그 프론트엔드에 반영
- 일일 1건 제한 (`run-blog-pipeline.ts`에서 체크)

## 4. 향후 개선 사항

### Phase 1 (현재)
- CLI 수동 실행 (`npm run pipeline:blog "주제"`)
- 주제는 사람이 선정

### Phase 2 (자동 주제 선정)
- 수집된 뉴스 기반 자동 주제 추천
- 검색 트렌드 API 연동 (Google Trends, Naver DataLab)
- 기존 포스트와의 중복/유사도 자동 체크

### Phase 3 (완전 자동화)
- cron/스케줄러로 매일 자동 실행
- 퀄리티 체크 자동화 (길이, 구조, 키워드 밀도)
- 성과 추적 (조회수, 체류시간) → 콘텐츠 전략 피드백 루프
